## Post 1 (X / Mastodon — lead post, all channels)

Incorporator v1.3.6: rate-limiting a host is now one call.

```python
register_host_penstock("api.example.com", rate_per_sec=50.0)
```

That's it. No `SustainedPenstock` import, no second symbol. (The instance
form still works if you prefer it.)

---

## Post 2 (X / Mastodon — follow-up)

Also in v1.3.6: the live retry wait now honors a server's `Retry-After`
header instead of just recording it after you've already been rejected.
One less thing to hand-wire on top of `httpx`.

And `architect.tune()` talks like a person now — every hint says what
happened, why it matters, and what to change. A clean run just says
"No tuning hints" instead of guessing.

## Post 3 (LinkedIn — longer form, combines both)

Incorporator v1.3.6 is out — a focused feature + DX release.

Highlights:

- `register_host_penstock(host, rate_per_sec=50.0)` replaces the old
  two-symbol `SustainedPenstock`/`BurstPenstock` setup with one call.
  The explicit instance form still works, unchanged.
- The live 429/503 retry wait now honors a server's `Retry-After` header
  during the retry itself, not just after the request has already failed.
- `watershed.json` and the Python API now resolve `@sidecar` conv_dict
  tokens the same way on both the CLI and Python entry paths.
- `architect.tune()`'s feedback is now plain-language: what happened, why
  it matters, what to change. A clean run honestly reports no hints
  instead of a false positive.

Less boilerplate, politer defaults, and a pipeline that tells you what to
fix. Details: docs/releases/v1.3.6.md
