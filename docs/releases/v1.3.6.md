# What's new in v1.3.6

Incorporator v1.3.6 is a focused feature + DX release since v1.3.5: less
ceremony setting up rate limiting, politer networking, and a pipeline that
tells you what to fix in plain English.

---

## 🪣 One call sets up host throttling

`register_host_penstock` now takes `rate_per_sec` (and optionally `burst`)
directly, so you don't need to reach for `SustainedPenstock` /
`BurstPenstock` yourself for the common case:

```python
# Before
from incorporator import register_host_penstock
from incorporator.io.penstock import SustainedPenstock
register_host_penstock("api.coingecko.com", SustainedPenstock(rate_per_sec=0.2))

# After
from incorporator import register_host_penstock
register_host_penstock("api.coingecko.com", rate_per_sec=0.2)
```

Add `burst=200` for a token-bucket ceiling; bare `rate_per_sec` gets you a
`SustainedPenstock` under the hood, mirroring the precedence
`resolve_penstock` already used. Pass both a `penstock` instance and
`rate_per_sec`/`burst` and it raises `TypeError` immediately — no silent
ambiguity about which one wins.

The explicit instance form is unchanged and fully backward compatible.
This is additive, not a replacement — reach for whichever reads better at
the call site.

## 🚦 Retry-After is honored during the live wait, not just after the fact

Previously, `Retry-After` was only consulted after a request had already
exhausted its retries and landed in `rejects`. During the live retry loop,
the wait callable used a plain exponential backoff regardless of what the
server asked for.

Now the live wait reads the same `Retry-After` parser and raises its floor
to match the header (bounded), so a 429/503 response gets the wait time
the server actually asked for — instead of getting re-hit early and
rejected anyway. One less thing to hand-roll on top of `httpx`.

The retry-attempt budget itself (`_make_http_stop`) is unchanged.

## 🧵 `watershed.json` and Python are the same pipeline now

`conv_dict` (and other token fields) in `watershed.json` can reference
`@sidecar` helper tokens on both entry paths. The CLI
(`incorporator tideweaver run` / `validate`) now resolves these the same
way the Python `load_watershed()` API already did — both routes share one
function, `merge_sidecar_extra_names`.

If you had an outflow-only `watershed.json` whose `conv_dict` referenced a
sidecar helper and it only worked from Python, it now works from the CLI
too, with no changes to your config.

## 🗣️ DX: the pipeline explains itself

`architect.tune()`'s post-run feedback has been rewritten in plain
language. Every hint now states what happened, why it matters, and what
to change — not framework jargon.

A static false-positive HIGH-severity retry-budget warning has also been
downgraded to informational, so a clean run reports honestly:

```
No tuning hints - all knobs look well-tuned for this run.
```

instead of crying wolf.

Related: `inc()` now warns when it's handed a callable or non-type
argument instead of silently passing values through unchanged — a common
typo for `calc(fn, "field")`. The `inc` / `calc` / `pluck` docs are now
consistently cross-referenced on one purpose axis: `inc` converts to a
type, `calc` applies a transform, `pluck` lifts a nested value.

## 🧹 Also in this release

A batch of correctness and observability fixes landed alongside the
above — payload-only passthrough, snapshot-wins watermark handling,
thread-safe schema registry access, AIMD growth-floor/hysteresis, and
`as_list()` row-aliasing among them. See `CHANGELOG.md` for the full list.

---

Upgrading is a drop-in — no config or call-site changes required to keep
existing behavior. Adopt the new shorthand and CLI parity where it saves
you typing.
